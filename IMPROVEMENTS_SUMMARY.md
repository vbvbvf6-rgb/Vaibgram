## 🔐 Безопасность и шифрование

### ✅ Добавлено сквозное шифрование сообщений
- **Файл:** `static/encryption.js` (новый)
- **Метод:** AES-GCM с WebCrypto API (встроенное в браузер)
- **Как работает:**
  - Каждая комната имеет свой ключ шифрования (основан на ID комнаты)
  - Ключ создаётся детерминированно, поэтому все участники используют один ключ
  - PBKDF2 для расширения ключа (100,000 итераций)
  - IV (инициализационный вектор) случайный для каждого сообщения
  - Сообщения кодируются в Base64 для передачи
- **Где используется:**
  - При отправке сообщения: `await MessageEncryption.encrypt(content, roomId)`
  - При получении сообщения: `await MessageEncryption.decrypt(content, roomId)`
  - При загрузке истории: все сообщения расшифровываются перед отображением
- **Функции:**
  - `encrypt()` - шифрует текст
  - `decrypt()` - расшифровывает текст
  - `isEncrypted()` - проверяет, зашифровано ли сообщение

### ✅ Усилена защита на бэкенде
- **Файл:** `backend/main.py` (обновлен)
- **Новый middleware: SecurityHeadersMiddleware**
  - `X-Content-Type-Options: nosniff` - предотвращает MIME sniffing
  - `X-Frame-Options: DENY` - предотвращает clickjacking
  - `X-XSS-Protection: 1; mode=block` - защита от XSS
  - `Content-Security-Policy` - строгая CSP политика
  - `Strict-Transport-Security` - HSTS для HTTPS
  - `Referrer-Policy` - контроль утечки referrer
  - `Cache-Control` - отключение кэширования для API
- **Улучшен Rate Limiter**
  - Max 150 requests/minute per IP
  - Автоматическое очищение старых записей

### ✅ Улучшена валидация входных данных
- **Файл:** `backend/routes.py` (обновлен)
- **Новые функции:**
  - `validate_username()` - проверка формата username (2-32 символа, алфанумерика+_-.)
  - `sanitize_username()` - удаление опасных символов и HTML тегов
  - `sanitize_content()` - экранирование HTML для сообщений
  - `validate_pin()` - проверка PIN
  - `validate_otp()` - проверка OTP кода
- **Где применяется:**
  - При регистрации - валидация и санитизация username
  - При обновлении профиля - санитизация
  - Защита от XSS инъекций и SQL injection (через SQLAlchemy ORM)

## 🐛 Известные ограничения

1. **Шифрование:** Все участники комнаты видят одно и то же сообщение (нет индивидуального шифрования)
